Problem 4 Pattern Recognition

Built from 20 real exam sets · 2011–2023

The one rule: You never invent a new algorithm. You translate the story into a graph (or other structure), pick the right known algorithm, and state the running time.

The exam says it every time: "use existing algorithms and data structures as much as you can."

Step 1 — Pick your algorithm

Are all edges the same cost / just counting steps?
YES → Use BFS — gives shortest path in unweighted graphs. O(V+E).
NO, edges have different weights (all ≥ 0) → Use Dijkstra. O((V+E) log V).

Are you asking "are A and B connected?" or "do they share a group?"
YES → Use Union-Find. O(α(N)) per operation ≈ constant.

Is there a forced ordering (prerequisites, increasing sequence, topological structure)?
YES → Build a DAG and use DAG Shortest Paths. O(V+E).

Is it about simulating a process (fire spreading, juggling, game turns)?
YES → Simulate directly, often with a priority queue or queue. O depends on problem.

Step 2 — Recognize the sub-question pattern

Problem 4 almost always has 2–3 sub-questions that build on each other:

Part a — Simplified version. "Assume unit weights" / "assume it's a straight line" / "assume all costs are 1."
→ Almost always: BFS or even a direct O(N) scan.

Part b — Add weights / generalize. "Now each edge has a cost" / "now the grid is not flat."
→ Almost always: Dijkstra.

Part c — Structural twist. "Now find the minimum X such that..." or "now the graph is a tree" or "describe a faster algorithm."
→ Could be: DAG SP, Union-Find, binary search + connectivity, or same algo + smarter graph.

Step 3 — Model the graph

Once you know the algorithm, you need to describe the graph. Answer these three questions:

  • What are the vertices? — Usually: each cell in a grid, each position/state, each person/location.
  • What are the edges? — Usually: you can move between adjacent cells, or take one action to go from one state to another.
  • What is the edge weight? — The cost or time of making that move. If everything costs 1: BFS. If costs vary: Dijkstra.
  • Is the graph directed or undirected? — If you can go both ways on every edge: undirected. If movement has a direction: directed.
  • How many V and E? — State this in terms of the problem's parameters (n, r, c, etc.) — this is what determines the running time.

Algorithm quick reference

BFS

Use when: all edges cost 1 (or 0/1 weights).

Gives: shortest path by number of steps.

Data structure: Queue (FIFO).

Time: O(V + E)
Dijkstra

Use when: edges have non-negative weights.

Gives: shortest weighted path from source.

Data structure: Min-priority queue.

Time: O((V+E) log V)
Union-Find

Use when: you need to track connected components.

Gives: "are u and v connected?" in near-constant time.

Use with: Kruskal's MST, infection spreading.

Time: O(α(N)) per op ≈ O(1)
DAG Shortest Path

Use when: graph has no cycles (forced ordering).

Gives: shortest path faster than Dijkstra.

Method: topological sort + relax in order.

Time: O(V + E)
All-pairs / multi-source

Run BFS or Dijkstra from every vertex.

Use when: you need distances between all pairs.

Examples: Wiener index, graph diameter, graph center.

Time: O(V·(V+E)) or O(V·(V+E) log V)

The most important patterns — memorise these

■ BFS — appears in ~12 of 20 exams

Grid navigation with unit cost · Maze / reachability · State-space search (lock, board game) · "Fewest steps" or "minimum number of moves"


■ Dijkstra — appears in ~11 of 20 exams

Weighted grid/graph · Travel time or distance with varying costs · "Shortest path" when weights ≥ 0


■ Union-Find — appears in ~5 of 20 exams

Connectivity questions · "Are these in the same group?" · Infection spreading · Kruskal's MST · Binary search + connectivity threshold


■ DAG shortest path — appears in ~4 of 20 exams

Prerequisites / ordering constraints · "Describe a faster algorithm" (when Dijkstra was already correct) · Temperature-ordered streets · Looped tree trick
